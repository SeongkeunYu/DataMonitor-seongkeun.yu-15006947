# PRD: DataMonitor PoC

## 1. 목적

JSON 형태로 저장된 데이터의 상태를 콘솔에서 실시간으로 조회할 수 있는 관리자 도구 개발을 위해, 핵심 파이썬 라이브러리들의 적합성을 검증한다.

---

## 2. 배경

- 운영 중인 시스템에서 데이터는 JSON 파일로 관리된다.
- 관리자가 별도 GUI 없이 터미널에서 데이터 상태를 빠르게 파악해야 한다.
- PoC를 통해 실제 도구 개발 전 기술 스택을 확정한다.

---

## 3. PoC 범위

### 3.1 핵심 기능

| 기능 | 설명 |
|------|------|
| JSON 데이터 로드 | 로컬 JSON 파일을 읽어 파싱 |
| 실시간 갱신 | 일정 주기로 데이터 변경 사항을 감지하고 화면 갱신 |
| 콘솔 UI 렌더링 | 테이블·트리 등 구조화된 형태로 데이터를 콘솔에 출력 |
| 필터/검색 | 키 또는 값 기준으로 데이터 필터링 |
| 변경 감지 하이라이트 | 이전 상태 대비 변경된 필드를 시각적으로 강조 |

### 3.2 PoC 대상 라이브러리

| 라이브러리 | 역할 | 검증 항목 |
|------------|------|-----------|
| `rich` | 콘솔 UI (테이블, 색상, 트리) | 렌더링 품질, 업데이트 성능 |
| `textual` | TUI 프레임워크 (위젯, 레이아웃) | 반응형 UI 구성 가능 여부 |
| `watchdog` | 파일 시스템 변경 감지 | JSON 파일 변경 이벤트 수신 |
| `deepdiff` | JSON 딕셔너리 diff 비교 | 변경 필드 정확도 |
| `pydantic` | JSON 스키마 검증 | 데이터 무결성 확인 |

---

## 4. 검증 시나리오

1. **기본 출력**: 샘플 JSON 파일을 로드해 콘솔 테이블로 출력한다.
2. **실시간 갱신**: JSON 파일을 수정하면 콘솔 화면이 자동 갱신된다.
3. **변경 감지**: 변경된 키/값이 하이라이트 또는 diff 형태로 표시된다.
4. **스키마 검증**: 잘못된 JSON 구조 입력 시 오류 메시지를 출력한다.
5. **필터링**: 특정 키워드로 JSON 데이터 내 항목을 필터링해 출력한다.

---

## 5. 성공 기준

- JSON 파일 변경 후 **1초 이내** 화면 갱신
- 중첩 JSON(depth 3 이상)도 구조를 유지하며 출력
- 변경된 필드를 색상 또는 기호로 명확히 구분
- 라이브러리 충돌 없이 단일 가상환경에서 동작

---

## 6. 제외 범위

- DB 연동, 네트워크 스트리밍 데이터는 이번 PoC에서 제외
- GUI(웹/데스크톱) 기반 도구는 대상 외
- 인증·권한 관리 기능 없음

---

## 7. 산출물

- 각 라이브러리별 샘플 코드 및 실행 결과 스크린샷
- 라이브러리 비교 평가표 (성능, 학습 비용, 유지보수성)
- 최종 기술 스택 선정 결과
